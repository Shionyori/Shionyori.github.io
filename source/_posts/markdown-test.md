本文逐一测试主题对 Markdown 各元素的渲染效果，方便后续写文章时参考。

## 标题层级

Hexo 文章标题建议从二级标题（`##`）开始，一级标题留给页面标题。

# H1 一级标题
## H2 二级标题
### H3 三级标题
#### H4 四级标题

## 文本样式

行内样式是技术写作中最常用的元素之一：

- **粗体** 用于强调关键词
- *斜体* 用于引入新术语
- ~~删除线~~ 标记过时内容
- `行内代码` 标注变量名、命令等

## 引用块

使用 `>` 引用外部资料或补充说明：

> 这是一段引用文字。
> 可以有多行。
>
> — 作者名

## 列表

### 无序列表

适合罗列相关但无次序的项目：

- 项目一
- 项目二
  - 嵌套项目
  - 另一个嵌套

### 有序列表

适合描述步骤或优先级：

1. 第一步
2. 第二步
3. 第三步

## 链接

Markdown 的超链接语法：`[文字](URL)`。

例如：[Hexo 官网](https://hexo.io/)。

## 表格

Markdown 表格适合呈现对比性数据：

| 功能 | 状态 | 备注 |
|------|:--:|------|
| 搜索 | ✅ | 本地搜索 |
| 暗色模式 | ✅ | 自动切换 |
| 响应式 | ✅ | 移动端适配 |
| 评论 | ❌ | 未开启 |

## 分割线

三个短横线 `---` 可以画出一条视觉分隔：

## 脚注

[^1]: 这是脚注的内容。
